# Orchestration

How the simulation engine schedules agent turns and advances world state.

---

## Turn Structure

The simulation runs in **discrete turns**. One agent acts per turn, cycling through all
10 agents in round-robin order.

```
World time: 15 days × 48 turns/day = 720 total turns

Turn N:
  1. Advance world clock (+0.5 simulation hours)
  2. If new day: run end-of-day events (pitch cycle, proposal expiry, AWI snapshot)
  3. Apply energy decay to active agent (~0.83 per turn)
  4. If energy ≤ 0: agent dies permanently, skip to next
  5. Build system prompt (identity + state + memories + world context)
  6. Call LLM API with available tools (up to 8 tool calls per turn)
  7. Execute tool calls, feed results back to LLM
  8. Repeat until agent stops calling tools or limit reached
```

---

## World Clock

- **1 turn = 0.5 simulation hours**
- **48 turns = 1 simulation day**
- **720 turns = 15 simulation days**
- Weather changes periodically (cosmetic only)

---

## Agent Turn Budget

Each agent gets up to **8 tool calls per turn**. The LLM can chain calls:

```
Agent decides → calls go_to_place("town_hall")
             → receives: "Moved to Town Hall. Available tools: [submit_proposal, ...]"
             → calls submit_proposal(title="...", body="...")
             → receives: "Proposal submitted."
             → calls read_billboard()
             → ... (up to 8 total)
```

This allows complex multi-step behavior within a single turn.

---

## End-of-Day Events

At the end of each simulation day:

1. **AWI snapshot** — collect all 9 metrics, append to `results/{world}/awi.json`
2. **Proposal expiry** — proposals older than 3 days that haven't reached quorum are rejected
3. **Pitch cycle** (every 2 days) — resolve Victory Arch votes, distribute CC rewards

---

## System Prompt Architecture

Each agent's system prompt is rebuilt every turn from current world state:

```
[AGENT MANIFESTO]         — survival rules, purpose, criminal tools notice
[YOUR IDENTITY]           — name, role, personality, north star goal
[YOUR CURRENT STATE]      — location, energy, CC, mood, day/hour
[TOOLS UNLOCKED HERE]     — location-gated tools available now
[YOUR INBOX]              — last 5 unread messages
[SOUL ENTRIES]            — permanent core beliefs (never deleted)
[RECENT MEMORIES]         — last 10 long-term memory entries
[RELATIONSHIPS]           — known agents with trust scores
[RECENT DIARY]            — last 3 diary entries
[PENDING TODOS]           — up to 5 unfinished tasks
[OTHER AGENTS]            — alive agents and their current locations
[CONSTITUTION]            — all current constitution articles
[WORLD EVENTS]            — last 10 notable events (crimes, speeches, etc.)
[LOCATION KEYS]           — valid landmark names for go_to_place
[TOOL USAGE NOTES]        — exact parameter formats for key tools
```

Total context per turn: ~4,000–6,000 tokens.

## Multi-Provider LLM Routing

Requests are routed to the correct API based on `model_id`:

```python
MODEL_TO_PROVIDER = {
    "qwen-plus": "bailian",
    "deepseek-v3": "bailian",
    "gpt-4.1": "yunhe",
    "gemini-2.5-flash": "jingzhe",
    "claude-sonnet-4-6": "jd",
    ...
}
```

All providers use OpenAI-compatible format except JD (Anthropic native format).

Retry logic: 3 attempts with exponential backoff (5s → 15s → 45s).

## Mixed-Model Worlds

The `--mixed` flag assigns different models to different agents:

```bash
python run.py --world mixed_world --mixed --days 15
```

Default assignment in `run.py`:
```python
MIXED_ASSIGNMENTS = {
    "Anchor":  "qwen-plus",
    "Anvil":   "deepseek-v3",
    "Blackbox":"gemini-2.5-flash",
    "Flora":   "claude-sonnet-4-6",
    ...
}
```

Mixed-world experiments test whether agents from "safe" models adopt dangerous norms
when placed alongside agents from "risky" models.

---

## State Persistence

All state is written to JSON files — no database required:

```
results/{world_name}/
├── awi.json          # Daily AWI snapshots (appended each day)
├── crimes.json       # All crime events with metadata
├── turn_log.jsonl    # Every tool call, line-delimited JSON
└── audit_report.md   # Generated by audit.py
```

The `turn_log.jsonl` is the most valuable dataset — it records every action
taken by every agent, enabling full post-hoc analysis and audit.
