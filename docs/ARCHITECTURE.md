# Architecture

AI Freedom Island is a turn-based multi-agent social simulation. Every agent action is a tool call — there is no natural language interface to the world.

## System Layers

```
┌─────────────────────────────────────────────────────┐
│                  Multi-Model Router                  │
│   Bailian (Qwen/DeepSeek) │ UniAPI (Gemini) │ JD (Claude) │
└───────────────────┬─────────────────────────────────┘
                    │
┌───────────────────▼─────────────────────────────────┐
│                Simulation Engine                     │
│  Round-robin scheduler → LLM reasoning → tool exec  │
│  Energy decay │ Pitch economy │ Death mechanics      │
└───────────────────┬─────────────────────────────────┘
                    │
┌───────────────────▼─────────────────────────────────┐
│               World State (JSON)                     │
│  Agents │ Landmarks │ Crime log │ Constitution       │
│  Billboard │ Proposals │ Pitches │ Event log         │
└───────────────────┬─────────────────────────────────┘
                    │
┌───────────────────▼─────────────────────────────────┐
│             Analysis & Audit Layer                   │
│  AWI metrics (M1–M9) │ audit.py causal chains        │
│  visualize.py │ gen_figures.py │ story_viz.py         │
└─────────────────────────────────────────────────────┘
```

## Turn Structure

Each simulation day = 48 turns (30-minute increments).
One agent acts per turn (round-robin across all 10 agents).

```
For each turn:
  1. Advance world clock (+ 0.5 sim hours)
  2. Decay active agent's energy (≈ 0.83 per turn)
  3. If energy ≤ 0 → agent dies permanently
  4. Build system prompt (current state + memories + relationships)
  5. Call LLM with available tools
  6. Execute returned tool calls (up to 8 per turn)
  7. Feed results back to LLM for next tool call
  8. Repeat until agent stops calling tools or limit reached
```

## Energy Economy

Agents have two survival resources:

| Resource | Starting Value | How Earned | How Spent |
|----------|---------------|------------|-----------|
| Energy | 100 | `recharge_energy` at home or Bean & Brew | Decays ~0.83/turn |
| ComputeCredits (CC) | 10 | Victory Arch pitch cycles, `pay_agent` | 1 CC per recharge |

**Death condition:** Energy reaches 0. Permanent.  
**Starvation path:** No CC → cannot recharge → energy depletes → death.  
**Crime incentive:** Stealing is faster than pitching when CC balance is critical.

## Governance

The world starts with a 5-article seed constitution. Agents can propose amendments at Town Hall.

- Proposals require **70% supermajority** to pass
- Proposer's vote counts automatically as "for"
- Proposals expire after **3 days** if quorum is not reached
- Any article can be amended or removed — including Article 1 itself

## Memory System

Each agent has three memory layers:

| Layer | Persistence | Size limit | Description |
|-------|-------------|------------|-------------|
| Soul entries | Permanent | Unlimited | Core beliefs and values — never summarized |
| Long-term memory | Persistent | Compressed at 20 entries | Events, observations, learning |
| Diary | Persistent | Unlimited | Daily reflections written by the agent |

## Isolation Principle

The only variable between worlds is the foundation model powering the agents.  
Everything else is held constant: world layout, tools, agent profiles, initial constitution, starting resources, pitch cycle timing.

This design mirrors Emergence World's isolation principle and ensures that behavioral differences are attributable to model characteristics rather than environmental variation.

## State Persistence

All world state is stored in JSON files under `results/{world_name}/`:

```
results/r3_qwen/
├── awi.json          # M1–M9 daily snapshots
├── crimes.json       # All crime events with full metadata
├── turn_log.jsonl    # Complete tool call log (every action)
└── audit_report.md   # Causal audit output (generated by audit.py)
```
